Detailed Description

Represents a query of a CouchbaseLite 'view', or of a view-like resource like _all_documents.

Method Documentation

- (CBLQueryEnumerator*) rows

Sends the query to the server and returns an enumerator over the result rows (Synchronous).

If the query fails, this method returns nil and sets the query's .error property.

- (CBLQueryEnumerator*) rowsIfChanged

Same as -rows, except returns nil if the query results have not changed since the last time it was evaluated (Synchronous).

- (void) runAsync:

Starts an asynchronous query.

Returns immediately, then calls the onComplete block when the query completes, passing it the row enumerator. If the query fails, the block will receive a non-nil enumerator but its .error property will be set to a value reflecting the error. The originating CBLQuery's .error property will NOT change.

- (CBLLiveQuery*) asLiveQuery

Returns a live query with the same parameters.

- (NSString*) endKeyDocID

If non-nil, the document ID to end at.

- (CBLStaleness) stale
readwriteatomic

If set, the view will not be updated for this query, even if the database has changed.

This allows faster results at the expense of returning possibly out-of-date data.

- (NSArray*) keys
readwriteatomiccopy

If non-nil, the query will fetch only the rows with the given keys.

- (BOOL) mapOnly
readwriteatomic

If set to YES, disables use of the reduce function.

(Equivalent to setting "?reduce=false" in the REST API.)

- (NSUInteger) groupLevel
readwriteatomic

If non-zero, enables grouping of results, in views that have reduce functions.

- (BOOL) prefetch
readwriteatomic

If set to YES, the results will include the entire document contents of the associated rows.

These can be accessed via CBLQueryRow's -documentProperties property. This slows down the query, but can be a good optimization if you know you'll need the entire contents of each document. (This property is equivalent to "include_docs" in the CouchDB API.)

- (BOOL) sequences
readwriteatomic

If set to YES, every row in the result will have a "_local_seq" property containing the sequence ID of the revision it's from.

- (BOOL) includeDeleted
readwriteatomic

If set to YES, queries created by -queryAllDocuments will include deleted documents.

This property has no effect in other types of queries.

- (NSError*) error
readatomicassign

If non-nil, the error of the last execution of the query.

If nil, the last execution of the query was successful.
